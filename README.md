Project Overview

This repository contains my work from CS-250 Software Development Lifecycle at Southern New Hampshire University.

During this course, I learned about Agile development, Scrum, user stories, testing, communication, and the Software Development Lifecycle (SDLC). I worked through several Scrum roles including Product Owner, Developer, Tester, and Scrum Master. The main project focused on developing a solution for SNHU Travel while applying Agile practices throughout the development process.

Reflection
How do I interpret user needs and implement them into a program? How does creating user stories help with this?

For me, it starts with understanding what the user is actually trying to accomplish. It's easy to focus on features, but if you don't understand the problem the user is trying to solve, it's hard to build something useful. User stories help because they make you think from the user's perspective. Instead of focusing only on technical requirements, you're focusing on what the user needs and why they need it. That makes it easier to build features that actually provide value.

How do I approach developing programs? What Agile processes do I hope to incorporate into my future development work?

I like breaking large problems into smaller pieces and then working through them one step at a time. One thing I learned from this course is that feedback should happen throughout development, not just at the end. In future projects, I'd like to continue using user stories, sprint planning, testing throughout development, sprint reviews, and retrospectives. Those practices help keep projects organized and make it easier to adjust when priorities or requirements change.

What does it mean to be a good team member in software development?

To me, being a good team member means communicating and making sure information is shared with the rest of the team. One thing that came up several times in this course was how many problems happen when people work in silos. If nobody communicates, work gets stuck and small issues can turn into bigger problems. A good team member shares information, helps when needed, and stays focused on the team's goals instead of only their own tasks. Communication and collaboration are just as important as technical skills.
